CRAP Design Audit — Before / After

Visual comparison of changes from Robin Williams' Contrast, Repetition, Alignment, Proximity audit

1. Filter Tab Active State High Impact

Reuses the inverted treatment from .reader-toolbar-seg.active. Active tab now inverts foreground/background for much stronger contrast.

After

2. Toast Notifications High Impact

Success and error toasts now reuse the --status-available-* and --status-dead-* design tokens for tinted backgrounds. Much easier to distinguish at a glance.

3. Reader Action Groups High Impact

Buttons split into 3 logical groups (status | audio | meta+destructive) with 1px vertical dividers. Easier to scan, parse, and find the right action.

Before
After

4. Card Delete Button Medium

The delete button in card action rows now uses the danger colour (#915550) at rest, not just on hover. Reuses the existing --danger variable.

Before
After

5. Save Form / Filter Bar Separation Medium

A 1px horizontal rule now separates the save form (adding content) from the filter bar (filtering content), clarifying two distinct functional zones.

After

Reader metadata now uses the same middle-dot separator pattern as article cards, reusing the .article-card-meta CSS convention.

10. Cleaned Design Tokens Low

Removed --shadow, --shadow-lg, and --shadow-up (all set to none) from the design system. Only --shadow-float remains — the single shadow actually used in the app.

11. Save Audio Button Consolidation Low

The bespoke .btn-save-audio class (24 lines of CSS) was replaced with the existing .btn .btn-sm .btn-secondary pattern. Visual result is nearly identical; hover/active/disabled states now match all other buttons.

12. Original Status Badge Placement Medium

The original status badge moved from between TagPicker and action buttons to between metadata and TagPicker. All informational elements now group together, all interactive elements group together.
